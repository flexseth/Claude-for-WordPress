# Component Definitions (MDX)

Use this folder for reusable component/resource definition pages.

## Authoring guidance

- Use one `.mdx` file per component/topic.
- Keep metadata in frontmatter when needed.
- Prefer plain Markdown unless you need JSX-powered composition.
- Keep examples deterministic and dependency-light.

## Minimal template

```mdx
---
title: Example Component
slug: /components/example-component
---

# Example Component

Describe purpose, props/options, usage examples, and known constraints.
```
