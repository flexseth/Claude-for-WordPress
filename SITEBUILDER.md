The purpose of this Claude project is to create WordPress tools with Claude Code.

# Project Overview
Work in progress features: plugin and block scaffolding and definitions
- Scaffold WordPress plugins based on user input
- Scaffold custom Gutenberg blocks using `@wordpress/create-block`
- Define common plugin types and features (settings page, shortcode, widget, custom post type, custom taxonomy, REST API endpoint, data store, block hooks, IAPI, HTML Tag Processor, block bindings, inner blocks, supports, API, integration, other)
- Include license file (GPLv2 or later)
- Reference necessary WordPress Core components, use examples when possible
- Update the plugin bootstrap and `readme.txt` file based on user input
- Create project plans with milestones and tasks
- Track project progress using GitHub Issues (Managed by Copilot)

# Scaffolding Process
Use `SCAFFOLDING.md` to gather user input and scaffold the plugin or block using the `create-block` package or standard WordPress plugin structure.
