## WP.org/plugins submission guidelines
The final step: Submitting your plugin to the WordPress.org Plugin Directory. Make sure to follow these guidelines to ensure a smooth review process.

- [ ] [] Follow the [detailed plugin guidelines](https://developer.wordpress.org/plugins/wordpress-org/detailed-plugin-guidelines/)
- [ ] [] Ensure the plugin meets all [plugin directory requirements](https://developer.wordpress.org/plugins/wordpress-org/detailed-plugin-guidelines/#plugin-directory-requirements)
- [ ] [] Adhere to the [plugin review team's best practices](https://developer.wordpress.org/plugins/wordpress-org/plugin-review/best-practices/)
- [ ] [] Verify the plugin does not contain any prohibited content or functionality as outlined in the [prohibited content guidelines](https://developer.wordpress.org/plugins/wordpress-org/detailed-plugin-guidelines/#prohibited-content)
- [ ] [] Ensure the plugin is compatible with the latest version of WordPress and follows [WordPress coding standards](https://developer.wordpress.org/coding-standards/wordpress-coding-standards/)
- [ ] [] Include a comprehensive `readme.txt` file that follows the [readme.txt guidelines](https://developer.wordpress.org/plugins/wordpress-org/how-your-readme-txt-works/)
